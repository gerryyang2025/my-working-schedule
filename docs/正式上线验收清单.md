# 正式上线验收清单

当前阶段：HTTP + 服务器公网 IP

本文用于正式上线当天逐项记录验收结果。当前阶段以公网 IP 的 HTTP 访问为准，不要求 HTTPS 验收通过。

## 部署前检查

- [ ] 已确认服务器 Node.js、npm、gcc、make、python3 等运行和构建依赖可用。
- [ ] 已确认防火墙和云安全组开放 TCP 80。
- [ ] 已确认 API 端口 3001 不直接暴露到公网。
- [ ] 已确认生产配置使用 SQLite，并设置管理员初始密码。
- [ ] 已确认数据目录 `/var/lib/my-working-schedule` 和备份目录 `/var/backups/my-working-schedule` 可创建或已存在。

## 部署执行

在源码目录执行：

```bash
./optools.sh deploy
```

记录部署结果：

- [ ] 前端构建完成。
- [ ] API 运行文件安装到 `/opt/my-working-schedule`。
- [ ] SQLite 初始化或升级完成。
- [ ] systemd 服务启动完成。
- [ ] Nginx 配置检查通过。
- [ ] logrotate dry-run 检查通过。

## HTTP + IP 访问验收

执行生产体检：

```bash
./optools.sh doctor
```

当 `./optools.sh doctor` 失败时，先查看检查项解释：

```bash
./optools.sh doctor guide
```

验收项：

- [ ] 公网访问 http://<server-ip>/ 正常打开首页。
- [ ] 公网访问 `http://<server-ip>/api/health` 返回健康状态。
- [ ] 当前阶段不要求 HTTPS 验收通过。
- [ ] `./optools.sh doctor` 中 HTTPS 检查如显示跳过，结论符合 HTTP + IP 阶段预期。

## 页面和账号验收

- [ ] 使用 `admin` 和已配置的管理员密码可以登录。
- [ ] 首页、排班、周统计、月结与奖金页面可以正常读取。
- [ ] 系统配置中的账号和审计日志页面可见。
- [ ] 新增 `viewer` 测试账号后，该账号可以登录。
- [ ] `viewer` 测试账号不能编辑排班。
- [ ] 测试账号可以修改自己的密码，修改后需要重新登录。

## 数据和备份验收

执行数据库检查：

```bash
./optools.sh data check
```

执行一次手动备份：

```bash
./optools.sh data backup
```

验收项：

- [ ] `./optools.sh data check` 输出完整性检查通过。
- [ ] 备份文件生成在 `/var/backups/my-working-schedule`。
- [ ] 备份文件大小和生成时间符合预期。
- [ ] 保存一条测试配置或排班后，SQLite 文件修改时间更新。

## 恢复演练验收

选择已确认可用的备份文件，在维护窗口内执行恢复演练：

```bash
./optools.sh app stop
CONFIRM_RESTORE=yes ./optools.sh data restore <backup-file>
./optools.sh data check
./optools.sh app start
./optools.sh app status
```

验收项：

- [ ] 恢复命令执行成功。
- [ ] 恢复后 `./optools.sh data check` 通过。
- [ ] API 服务重新启动成功。
- [ ] 恢复后首页和关键业务页面可访问。
- [ ] 恢复后管理员账号可登录。
- [ ] 恢复演练记录已填写到 [备份恢复演练记录模板.md](./备份恢复演练记录模板.md)。
