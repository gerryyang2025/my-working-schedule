
# Web 网站样式风格建议方案

## 文档状态

本文与当前执行计划的前端方向保持一致：一期采用 Vue 3 + Vite + Element Plus 实现表格密集型管理界面，后端为 Express API + SQLite 文件数据库。文中涉及奖金分配仪表盘、Go/Gin/etcd 等内容属于后续扩展或正式架构方向，不进入一期实现。

## 1. 整体视觉风格：医用专业与极简主义 (Healthcare Professional & Minimalist)
*   **配色方案**：
    *   **主色调**：建议采用**深蓝色（#2C3E50）**或**医疗蓝（#007BFF）**作为导航栏和主要按钮颜色，传递专业、严谨和信任感。
    *   **背景色**：使用**浅灰或米白色（#F8F9FA）**，减少长时间排班操作带来的视觉疲劳。
    *   **警示色（核心需求）**：根据源文件要求，**周末及法定节假日**背景或文字必须设为**醒目的红色**，并显示节日名称（如“端午节”）；对于“夜”、“价”等特殊班次，也应使用**红色**字体以示区分。
*   **字体选择**：优先选用无衬线字体（如 PingFang SC, Microsoft YaHei），确保在复杂的表格数据中依然保持高清晰度。

## 2. 核心布局建议：交互式排班矩阵
*   **固定表头与首列 (Sticky Headers & Columns)**：
    *   排班表应采用响应式设计。当人员名单较多或日期跨度较长时，左侧的**工号/人员姓名**列（如“段鸿露 000228”）和顶部的**日期/星期**行应保持固定不动，方便用户随时核对排班位置。
*   **紧凑型网格设计**：
    *   由于排班信息密度大，单元格应设计得较为紧凑，但保留足够的点击区域。
    *   **悬浮高亮**：当鼠标划过某一行或某一列时，该行/列应有轻微的背景色变化，辅助护士长进行横纵向定位。

## 3. 功能交互组件：快捷操作面板
*   **班次选择侧边栏/浮窗 (Quick Shift Selector)**：
    *   根据视频中的“一点击就可输入”需求，建议在屏幕右侧或底部设置一个**班次快捷悬浮窗**。
    *   将常用班次（如 A1组长、P1、N1）和不常用班次（如进修、病假）分类排列。
    *   **一期操作逻辑**：用户先选择班次画笔，再点击人员/日期单元格快速填入；也可以点击单元格打开弹窗，手动设置最多两个班次。数据通过 Express API 写入 SQLite。
*   **全屏切换与搜索**：
    *   提供一键**全屏显示**按钮，最大化排班视野。
    *   在顶部设置**全局搜索框**，支持按姓名或工号快速定位到特定员工。

## 4. 报表与打印样式 (Print-Ready Design)
*   **打印预览模式**：
    *   针对“打印排班申请”的需求，需要专门设计一套 **CSS Print 样式**。
    *   打印视图应自动隐藏侧边栏、按钮等 UI 元素，仅保留纯净的表格结构，并自动调整为 A4 横向布局，确保工号、姓名、总天数及总系数等信息完整展示。
*   **实时核算浮层**：
    *   在表格末尾的“出勤总天数”和“总系数”列，可以设计成实时变动的数值小组件。当护士长修改一个班次时，该行末尾的绩效数值应有轻微的闪烁动画，提示数据已完成自动核算。

## 5. 后续奖金分配仪表盘 (Bonus Dashboard)
*   **沉浸式输入框**：
    *   奖金分配不进入一期执行计划。后续实现时，可设计一个醒目的“奖金总额”输入组件。当输入金额后，下方列表以**卡片式或平铺列表**形式实时展示每位员工计算后的奖金数额，配合进度条或占比环状图，让分配逻辑一目了然。

# 总结
该风格方案旨在平衡**操作的高效性**与**数据的可读性**。通过红色警示色与蓝色专业色调的配合，既能满足源文件中提到的视觉区分需求，又能适配一期轻量 Web 应用和后续正式架构演进。

**可以选择以下任意一种具体的 UI 框架风格来实现：**
*   **Element Plus (Vue 3)**：风格更贴近企业管理系统，表格功能极其强大，适合此系统的多维度统计。一期执行计划采用该方案。
*   **Ant Design (React/Vue)**：设计感更强，组件细节丰富，适合追求界面精致度的用户。
