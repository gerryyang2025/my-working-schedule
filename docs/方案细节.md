# 一期落地细节与执行计划对齐说明

## 文档状态

本文用于记录早期隐性需求在当前执行计划中的处理结果。当前一期只实现排班管理闭环；绩效奖金、月结锁定、完整账号体系和 etcd 运维作为后续扩展。

## 1. 权限与多角色协作

已确认的一期规则：

*   系统默认所有人可查看。
*   输入管理密码后进入编辑模式。
*   护士长和指定管理员可以维护人员、班次、节假日和排班。
*   护士、文员默认只读，除非被指定为管理员。

一期不做完整账号登录、多端会话和操作审计。后续可扩展为正式角色权限体系。

## 2. 人员类型与核算规则

已确认的一期规则：

*   护士、文员、护士长都显示在排班表中，也都可以被安排班次。
*   文员和护士按相同规则参与出勤、加班和总系数统计。
*   护士长按相同排班规则统计出勤和加班，但绩效系数单独核算，不并入普通人员绩效系数。

因此，一期数据模型需要把“人员类型”和“是否有编辑权限”拆开处理。

## 3. 节假日维护

已确认的一期规则：

*   系统内置默认节假日。
*   管理员可以新增、修改、删除节假日。
*   节假日配置包含日期、名称和是否影响满勤。
*   影响满勤的节假日会扣减当周满勤标准。

一期不接外部节假日 API。后续可根据医院制度增加自动更新或年度批量导入。

## 4. 历史数据与月结

一期早期只验证日常排班和周统计流程；当前系统已演进为 SQLite 运行时存储，并已实现月结锁定、奖金快照和操作审计等正式化能力。

后续当奖金分配进入实现范围时，需要补充：

*   月结锁定。
*   历史查询。
*   操作审计。
*   数据备份。

## 5. 打印格式

一期打印采用 CSS Print：

*   月排班表打印用于张贴和归档。
*   周统计表打印用于核算确认。
*   打印时隐藏按钮、侧边栏、配置抽屉等操作控件。
*   页面按 A4 横向输出，保留人员、日期、班次、节假日、出勤班次数、加班班次数和总系数。

如医院有固定签字栏或报批模板，后续可追加专用打印模板或 PDF 输出。

## 6. 排班冲突与校验

一期直接阻止：

*   未进入编辑模式修改数据。
*   单人单日超过两个班次。
*   同一单元格保存重复班次。
*   保存不存在或已禁用的班次。
*   保存不存在的人员。
*   节假日日期重复。

一期提示但不强制阻止：

*   连续多日上班。
*   周满勤标准被节假日扣减到异常低值。

夜班后必须休息、连续 7 天上班等更复杂规则，后续可进入规则引擎。

## 7. 存储与运维

当前运行时使用 SQLite 文件数据库。前端只通过 API 访问数据，后续可以在保持 API 结构稳定的前提下评估 PostgreSQL 或其他集中式存储。

后续正式化时需要补充：

*   etcd 快照或其他备份机制。
*   数据库 schema 升级脚本。
*   操作日志。
*   故障恢复方案。

## 8. 当前执行计划覆盖

当前执行计划覆盖以下内容：

*   Vue 3 + Vite + Element Plus 前端。
*   Express API。
*   SQLite 文件数据库。
*   日期和自然周工具。
*   排班校验。
*   周统计核算。
*   人员、班次、节假日维护。
*   月排班表和周统计表打印。
*   Vitest、Supertest、Playwright 验证。
