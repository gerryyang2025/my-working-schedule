# 工作台 Tab 与范围奖金试算设计

## 背景

当前首页同时展示排班表、班次画笔、周统计和月结与奖金信息。随着月结、奖金分配和打印能力增加，首页信息密度变高，移动端浏览和日常排班操作容易受到干扰。

本设计覆盖下一阶段优化：

- 首页改为工作台 Tab 结构。
- 月结与奖金支持整月范围的临时汇总和奖金试算。
- 月结与奖金人员明细增加累计加班班次。
- 排班表人员列、班次文字和统计面板样式做视觉优化。

## 目标

- 减少首页默认信息量，让排班、周统计、月结与奖金分区浏览。
- 保留手机端和 PC 端快速点击格子排班的操作方式。
- 支持用户选择开始月份和结束月份，查看跨月临时汇总与奖金试算。
- 在月结与奖金明细中增加累计加班班次，便于核对排班负荷。
- 缩短排班表人员列宽，提升一周 7 天排班表的可视空间。
- 排班格班次显示去掉外框，只保留更清晰的彩色文字。
- 统一周统计和月结与奖金的字体大小、标题层级和表格风格。

## 非目标

- 不支持跨月正式月结。
- 不保存跨月范围试算结果。
- 不因跨月范围试算锁定任何排班。
- 不改变单月正式月结的锁定和取消逻辑。
- 不改变护士长绩效单独核算规则。
- 不实现账号、审计或正式部署能力。

## 首页 Tab 信息架构

首页改为工作台 Tab 结构。

PC 端使用左侧竖向 Tab，右侧展示当前 Tab 内容。手机端使用顶部横向 Tab，方便窄屏切换。

Tab 包含：

- `排班`：班次画笔 + 当前选中日期所在周的 7 天排班表。
- `周统计`：当前选中日期所在周的周统计。
- `月结与奖金`：单月月结、自定义整月范围汇总、奖金试算。

顶部工具栏保留：

- 日期选择。
- 上一周、本周、下一周。
- 当前周范围显示。
- 输入管理密码/编辑模式。
- 配置。
- 打印周表。
- 打印月表。
- 全屏。

切换 Tab 不改变当前选中日期。排班 Tab 仍支持通过班次画笔点击格子快速排班。

## 月结与范围试算

`月结与奖金` Tab 顶部增加两个按月选择框：

- 开始月份。
- 结束月份。

默认值：

- 开始月份 = 当前选中日期所在月份。
- 结束月份 = 当前选中日期所在月份。

### 单月模式

当开始月份等于结束月份时，展示现有单月月结能力。

未月结时：

- 展示实时月度汇总。
- 输入奖金总额。
- 展示奖金分配试算。
- 支持确认月结。
- 确认月结前继续弹出二次确认。
- 确认后锁定该月排班。

已月结时：

- 展示月结快照。
- 展示月结时间。
- 展示奖金总额。
- 展示奖金分配结果。
- 支持取消月结。
- 取消后该月恢复可编辑。

### 多月范围试算

当开始月份小于结束月份时，展示临时范围汇总与奖金试算。

范围按整月包含：

- 例如 2026-06 至 2026-07 表示 2026-06-01 至 2026-07-31。

数据来源：

- 已月结月份使用月结快照。
- 未月结月份使用实时排班汇总。

试算行为：

- 合并范围内人员汇总。
- 按合并后的普通人员总系数计算奖金分配。
- 护士长继续排除普通奖金池。
- 显示“临时试算，不会保存或锁定排班”提示。
- 不显示确认月结按钮。
- 不显示取消月结按钮。
- 不保存任何跨月结果。
- 不锁定任何排班。

当开始月份大于结束月份时：

- 显示提示，引导用户调整范围。
- 不计算汇总和奖金试算。

## 累计加班班次

在 `月结与奖金` 的人员明细中新增列：`累计加班班次`。

口径：

- 按班次统计，不按自然日统计。
- 一天两个班次，出勤计 2 个班次。
- 每个自然周单独计算加班。
- 对于范围边界周，只统计落在选定范围内的日期。
- 边界周满勤标准按范围内有效日期折算，并扣除范围内影响满勤的节假日。
- 每周加班 = `max(0, 范围内出勤班次 - 范围内满勤标准)`。
- 累计加班班次 = 范围内各周加班班次合计。
- 护士长和文员也统计出勤与加班班次。
- 护士长绩效系数仍显示“护士长绩效单独核算”。

显示位置：

- 单月月结表在 `出勤班次` 后增加 `累计加班班次`。
- 多月范围试算表同样显示 `累计加班班次`。
- 月结快照和月表打印同步显示该列。

## 数据与计算设计

新增或扩展范围汇总计算能力。

建议在计算层增加可复用的范围汇总函数：

- 输入：应用数据、开始日期、结束日期。
- 输出：范围开始日期、范围结束日期、总天数、节假日名称、人员汇总 rows。
- rows 包含：人员、类型、出勤班次、累计加班班次、总系数、排除原因。

单月汇总可以复用该范围汇总能力，避免月度和范围逻辑分叉。

多月范围合并时：

- 对已月结月份，优先使用月结快照 rows。
- 对未月结月份，使用实时范围汇总 rows。
- 同一人员按 `staffId` 合并。
- 出勤班次累加。
- 累计加班班次累加。
- 普通人员总系数累加。
- `coefficientTotal = null` 的人员保持单独核算，不参与普通奖金池。

月结快照需要扩展保存 `overtimeShifts`，用于已月结月份后续范围试算和打印保持冻结口径。

旧数据兼容：

- 旧的 `MonthlySettlementRow` 没有 `overtimeShifts` 时，加载时可以补默认值 `0`，或在规范化阶段按历史快照可用数据计算安全默认值。
- 第一版建议补默认值 `0`，避免旧快照加载失败；新快照保存完整字段。

## 前端交互设计

`月结与奖金` Tab 的顶部区域包含：

- 开始月份选择。
- 结束月份选择。
- 当前模式提示：单月月结 / 临时范围试算。

单月模式保留现有操作：

- 奖金总额输入。
- 确认月结。
- 取消月结。
- 月结状态。

多月范围模式：

- 奖金总额输入。
- 奖金分配试算。
- 不显示确认月结和取消月结。
- 显示说明：临时试算，不会保存或锁定排班。

奖金总额输入在单月和多月都可用：

- 单月用于正式月结前试算。
- 多月用于临时奖金分配试算。

## 排班表视觉优化

人员列宽缩短：

- PC 从当前约 `132px` 调整到约 `108px-116px`。
- 手机从当前约 `100px` 调整到约 `84px-88px`。
- 姓名优先显示。
- 工号缩小显示。
- 必要时换行。

排班格班次显示：

- 去掉班次外框。
- 不再显示带边框 chip。
- 只保留班次文字，例如 `办公`、`A1`、`N1`。
- 文字使用班次颜色。
- 字体适当放大，增强可读性。
- 一天两个班次时上下排列，保持格子尺寸稳定。

## 统计面板视觉统一

`周统计` 和 `月结与奖金` 使用统一统计面板风格：

- 标题字号一致。
- 说明文字颜色和字号一致。
- 表格字号和边框一致。
- 移动端紧凑行风格一致。
- 卡片边框、内边距、背景保持统一。

月结与奖金可以保留业务专属状态标签，例如“已月结”“未月结”“临时试算”，但标签风格应和周统计的说明信息协调。

## 打印与 PDF

单月月表打印继续支持：

- 月排班表。
- 月度汇总。
- 已月结时奖金分配快照。

新增 `累计加班班次` 后：

- 月度汇总打印表增加该列。
- 已月结奖金分配快照表增加该列。

多月范围试算第一版不新增独立打印模板。

## 错误处理

- 开始月份大于结束月份时提示调整范围，不计算。
- 多月范围试算不显示正式月结操作，避免误以为可以跨月锁定。
- 奖金总额无效时不计算奖金分配。
- 普通人员总系数为 0 时显示现有提示，不允许正式月结。
- 范围内部分月份已月结时，页面提示这些月份使用快照数据。

## 测试范围

新增或扩展测试：

- 范围汇总计算：单月、跨月、边界周、节假日扣减、一天两个班次。
- 累计加班班次：按范围内自然周拆分计算。
- 已月结月份 + 未月结月份混合范围：快照和实时数据合并。
- 月结快照保存 `overtimeShifts`。
- 旧月结快照兼容缺失 `overtimeShifts`。
- 奖金分配试算包含累计加班班次列。
- 单月模式仍可确认/取消月结。
- 多月范围模式不显示正式月结操作。
- Tab 切换不改变当前选中日期。
- 排班 Tab 保持快速点击格子排班。
- 移动端 Tab 和表格布局可用。
- 打印/PDF 月表包含累计加班班次。

## 实施建议

建议分阶段实现：

1. 先做工作台 Tab 和排班表视觉优化。
2. 再扩展范围汇总计算和累计加班班次。
3. 最后接入月结与奖金的范围试算 UI 和打印/PDF 列扩展。

这样可以把低风险 UI 改动和高风险统计口径改动分开验证。
