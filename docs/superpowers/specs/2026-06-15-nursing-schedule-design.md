# 国际医学部护理排班管理系统一期设计

## 背景

国际医学部护理排班目前需要覆盖人员排班、班次颜色识别、日期与周查询、全屏展示、打印输出，以及按周自动统计出勤班次、加班班次和总系数。第一期目标是先做一个轻量正式版，把护士长和指定管理员的真实排班流程跑通，并为后续绩效、奖金分配和 Go/Gin/etcd 正式架构预留边界。

本设计基于仓库中的 `docs/产品需求.md`、`docs/技术方案.md`、`docs/方案细节.md`，并按需求讨论结果收敛一期范围。

## 一期范围

一期实现一个前端 Web 应用加简单后端的排班管理系统。系统默认所有人可查看；输入管理密码后进入编辑模式，护士长和指定管理员可以维护人员、班次、节假日和排班记录。

核心能力包括：

- 月视图排班表。
- 日期选择与周次选择，快速定位自然周。
- 班次画笔快速填班。
- 单元格弹窗精细编辑，支持单人单日最多两个班次。
- 班次颜色区分。
- 全屏显示。
- 月排班表打印。
- 周排班明细与统计打印。
- 按周自动统计出勤班次数、加班班次数和总系数。
- 人员、班次、节假日的页面维护。
- 示例初始数据，正式使用前可由管理员调整。

一期暂不实现奖金分配、月结锁定、完整账号登录、审计日志、流程审批、外部节假日接口和 etcd 存储。

## 用户与权限

系统区分展示、被排班、编辑权限三个概念。

人员类型包括护士、文员、护士长。三类人员都显示在排班表中，也都可以被安排班次。

护士和文员按相同规则参与出勤、加班和总系数统计。护士长排班规则与其他人一样，参与出勤和加班统计，但护士长绩效系数不纳入普通人员绩效核算，后续单独处理。

编辑权限不由人员类型直接决定。护士长和被授予管理员权限的人员可以进入编辑模式；护士、文员默认只读，除非被指定为管理员。第一期不做完整账号体系，采用管理密码进入编辑模式。

## 页面与模块

### 排班工作台

排班工作台是主页面，包含年月选择、日期选择、周次选择、全屏按钮、打印按钮和编辑模式状态。

排班表以人员为行、日期为列。表头显示日期、星期和节假日名称。周末和节假日需要用明显的视觉样式标识。

班次录入支持两种方式：

- 先选择班次，再点击格子快速填写。
- 点击单元格打开编辑弹窗，调整主班次或添加第二班次。

普通点击默认替换主班次。一天两个班次属于少数情况，只能在编辑弹窗中手动添加。单人单日最多保存两个班次。

### 周统计面板

选中某一自然周后展示周统计面板。周范围固定为周一到周日。

周统计展示：

- 人员。
- 出勤班次数。
- 满勤标准。
- 加班班次数。
- 总系数。
- 本周节假日扣减说明。
- 护士长绩效系数单独核算说明。

周统计用于审核当周排班和打印周统计表。

### 人员管理

人员管理维护：

- 工号。
- 姓名。
- 人员类型：护士、文员、护士长。
- 是否指定管理员。
- 是否启用。
- 排序。

禁用人员保留历史记录，但不应参与新的排班选择。

### 班次管理

班次管理由护士长或管理员维护。系统内置示例班次和默认颜色，正式使用前可以修改。

班次字段包括：

- 名称。
- 简称。
- 颜色。
- 是否计出勤。
- 系数。
- 是否启用。
- 排序。

班次颜色默认由系统给出，管理员可调整。排班、统计和打印都使用同一套班次配置。

### 节假日管理

系统内置默认节假日数据，管理员可新增、修改、删除节假日。

节假日字段包括：

- 日期。
- 名称。
- 是否影响满勤。

节假日名称需要显示在排班表表头。影响满勤的节假日用于扣减当周满勤标准。

## 查询与打印

系统提供两类查询入口：

- 年月选择，用于查看整月排班表。
- 日期选择和周次选择，用于快速定位自然周。

打印支持两类输出：

- 月排班表打印：保留人员、日期、星期、节假日、班次和颜色标识，适合张贴和归档。
- 周统计表打印：包含周排班明细、满勤标准、节假日扣减说明、每人出勤班次数、加班班次数、总系数和护士长绩效单独核算说明。

全屏显示用于大屏查看和会议讨论，不改变数据。

## 数据与存储

一期采用前端 Web 加简单后端和 JSON 文件存储。前端通过 API 读写数据，后端负责读写本地 JSON 文件。`data/app-data.json` 作为仓库种子数据，开发运行时默认写入已忽略的 `data/app-data.local.json`，也可以通过 `SCHEDULE_DATA_PATH` 指定其他数据文件。后续可在保持 API 结构稳定的前提下替换为 Go/Gin/etcd。

数据分为：

- `staff`：人员资料。
- `shifts`：班次配置。
- `holidays`：节假日配置。
- `scheduleEntries`：排班记录。
- `settings`：管理密码、默认满勤班次数、版本等系统配置。

`scheduleEntries` 以日期和工号定位，每条记录最多保存两个班次。记录中保存班次标识，展示和统计时再读取当前班次配置。

数据写入时需要校验：

- 未进入编辑模式不能修改。
- 人员必须存在。
- 班次必须存在且启用。
- 单人单日最多两个班次。
- 同一个单元格不能保存重复班次。
- 节假日日期不能重复。

## 核算规则

一期核算全部按班次口径，不按自然日工时。

每个排班单元格最多两个班次。如果两个班次都配置为计出勤，则出勤班次数为 2。休、病假、产假等是否计出勤，由班次配置决定。

周范围固定为周一到周日。默认每周满勤标准为 5 个计出勤班次。当周存在配置为影响满勤的法定节假日时，满勤标准按节假日数量扣减。例如当周有 1 个影响满勤的节假日，满勤标准为 4；某人实际计出勤班次数为 6，则加班班次数为 2。

加班班次数计算公式：

```text
加班班次数 = max(0, 计出勤班次数 - 当周满勤标准)
```

总系数按班次系数累加。护士和文员的总系数正常纳入普通统计。护士长参与出勤和加班统计，但护士长绩效系数不纳入普通绩效核算；界面和打印中需要明确标注护士长绩效单独核算。

缺勤差额和奖金分配不作为一期核心输出，后续扩展。

## 异常处理

系统直接阻止以下情况：

- 未进入编辑模式修改数据。
- 单人单日超过两个班次。
- 选择已禁用班次。
- 保存不存在的人员或班次。
- 节假日日期重复。

系统提示但不强制阻止以下情况：

- 连续多日上班。
- 某周没有配置节假日但用户预期应有节假日。
- 某周满勤标准被扣减到异常低值。

## 验证口径

一期至少验证以下场景：

- 新增或修改班次后，排班表可选择该班次。
- 每人每天最多两个班次。
- 普通点击替换主班次。
- 编辑弹窗可添加第二班次。
- 按日期和周次能定位到同一自然周。
- 节假日能显示在表头。
- 影响满勤的节假日会扣减当周满勤标准。
- 超过满勤标准才产生加班班次数。
- 护士和文员按同样规则统计。
- 护士长出勤和加班可统计，但绩效系数排除或标注单独核算。
- 月打印视图与月排班表一致。
- 周打印视图与周统计结果一致。
