# 奖金分配与月结锁定设计

## 背景

系统已经具备周排班、周统计、月度汇总和月表汇总打印能力。下一阶段需要在月度汇总基础上完成奖金分配，并在确认后锁定当月排班，避免结算后历史数据被直接修改。

本设计只覆盖第二阶段：奖金分配和月结锁定。完整账号体系、审计日志和正式部署继续作为后续阶段。

## 目标

- 基于已实现的月度汇总计算普通人员奖金分配。
- 按月总系数分配奖金，护士长排除普通奖金池。
- 保存月结快照，包含奖金总额、人员月度汇总、分配金额和结算时间。
- 月结后锁定对应月份排班记录，禁止直接修改。
- 支持取消月结，取消后该月恢复可编辑，并可重新生成月结快照。
- 已月结月份展示快照结果；未月结月份展示实时月度汇总和试算结果。

## 非目标

- 不实现完整账号登录和角色权限。
- 不实现审计日志。
- 不实现审批流程。
- 不实现正式生产部署。
- 不处理护士长绩效奖金的单独算法，只继续显示“护士长绩效单独核算”。
- 不引入月满勤标准或月加班班次。

## 奖金分配口径

奖金分配按月总系数进行。

参与普通奖金池的人员：

- `coefficientTotal` 为数字的人员参与分配。
- `coefficientTotal = null` 的人员不参与分配，当前主要是护士长。
- 月总系数为 `0` 的人员保留在结果中，但分配金额为 `0.00`。

分配公式：

```text
个人奖金 = 奖金总额 × 个人月总系数 / 普通人员月总系数合计
```

金额保留两位小数。为了避免四舍五入造成合计金额与奖金总额相差几分钱，最后一个参与分配且月总系数大于 0 的人员吸收尾差。

当普通人员月总系数合计为 `0` 时：

- 所有人分配金额为 `0.00`。
- 界面提示“普通人员月总系数合计为 0，无法按系数分配奖金”。
- 不允许确认月结。

奖金总额要求：

- 必须大于或等于 `0`。
- 保存和快照中统一保留两位小数。
- 奖金总额为 `0` 时允许试算，但确认月结前需要明确展示所有分配金额为 `0.00`。

## 月结锁定口径

月结以月份为单位，月份标识使用 `YYYY-MM`，例如 `2026-06`。

确认月结后：

- 保存该月奖金总额。
- 保存该月月度汇总快照。
- 保存每个人的分配金额。
- 保存结算时间。
- 保存结算状态为 `locked`。
- 禁止新增、修改、清空该月任何排班记录。

月结锁定只限制排班记录，不限制人员、班次和节假日配置维护。历史月结快照不随后续配置变化而变化。

取消月结后：

- 删除或标记该月月结快照为取消状态。
- 该月排班恢复可编辑。
- 重新确认月结时基于当前实时数据生成新的快照。

第一版采用删除式取消：取消后从 `monthlySettlements` 中移除该月份记录，避免在没有审计日志前维护多版本历史。

## 数据模型

在 `AppData` 中新增 `monthlySettlements`。

```ts
export interface MonthlySettlementRow {
  staffId: string;
  staffName: string;
  staffType: StaffType;
  attendanceShifts: number;
  coefficientTotal: number | null;
  coefficientExcludedReason: string;
  bonusAmount: number;
  bonusExcludedReason: string;
}

export interface MonthlySettlement {
  id: string;
  month: string;
  monthStart: string;
  monthEnd: string;
  totalDays: number;
  bonusPool: number;
  coefficientTotal: number;
  settledAt: string;
  rows: MonthlySettlementRow[];
}
```

字段说明：

- `bonusPool`：当月普通奖金池总额。
- `coefficientTotal`：参与普通奖金池人员的月总系数合计。
- `bonusAmount`：个人分配金额，护士长和排除人员为 `0`。
- `bonusExcludedReason`：护士长显示“护士长绩效单独核算”，普通人员为空。

## 计算与快照

新增 `calculateBonusAllocation(monthlySummary, bonusPool)`：

- 输入实时月度汇总和奖金总额。
- 输出试算结果，不直接写入存储。
- 按月总系数分配普通奖金池。
- 护士长不参与普通奖金池。
- 对最后一个正系数参与者做尾差修正。

新增 `createMonthlySettlement(monthlySummary, bonusPool, settledAt)`：

- 基于月度汇总和奖金试算结果生成月结快照。
- `settledAt` 由服务端写入当前时间。
- 快照保存后即作为已月结月份的展示来源。

## 后端接口

在现有简化 API 下新增：

- `PUT /api/data/monthly-settlement`
  - 需要管理员 token。
  - 请求体包含 `month` 和 `bonusPool`。
  - 服务端根据当前数据重新计算该月月度汇总并生成快照。
  - 如果该月已月结，返回错误。
  - 成功后返回最新 `AppData`。

- `DELETE /api/data/monthly-settlement/:month`
  - 需要管理员 token。
  - 取消指定月份月结。
  - 成功后返回最新 `AppData`。

排班保存接口 `PUT /api/data/schedule-entry` 增加锁定检查：

- 根据请求日期得到 `YYYY-MM`。
- 如果该月存在月结记录，则拒绝保存。
- 错误信息使用“该月份已月结，不能修改排班”。

## 前端交互

新增“月结/奖金”入口，第一版放在系统配置抽屉或主页面工具区的轻量按钮中。进入后展示当前选中日期所在月份。

未月结月份展示：

- 月度汇总实时结果。
- 奖金总额输入框。
- 奖金分配试算表。
- “确认月结”按钮。

已月结月份展示：

- “已月结”状态。
- 月结时间。
- 奖金总额。
- 快照分配结果。
- “取消月结”按钮。

确认月结前需要二次确认，提示该月排班会被锁定。

## 打印与 PDF

月表打印继续显示实时月度汇总或月结快照：

- 未月结月份：显示实时月度汇总，不显示奖金分配金额。
- 已月结月份：在月度汇总下方显示奖金分配快照，包括奖金总额、月结时间和个人分配金额。

第一版不新增独立奖金打印模板。

## 错误处理

- 未进入编辑模式不能确认或取消月结。
- 已月结月份不能再次确认月结。
- 未月结月份取消月结返回明确错误。
- 已月结月份保存排班返回明确错误。
- 普通人员月总系数合计为 `0` 时不能确认月结。
- 奖金总额无效时不能确认月结。
