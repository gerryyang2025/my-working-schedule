# 月度汇总与月表汇总打印设计

## 背景

当前系统的一期能力已经覆盖周排班工作台、周统计、月排班打印和移动端 PDF 输出。现有月表只展示整月排班明细，不展示月度出勤和绩效系数汇总。后续奖金分配、月结锁定、审计和正式部署都需要稳定的月度核算基础，因此先补齐独立的月度汇总能力。

本设计只覆盖第一阶段：月出勤班次、月总系数和月表汇总打印。奖金分配、月结锁定、账号权限、审计日志和正式部署继续作为后续阶段。

## 目标

- 根据当前打印月份自动统计每个人的月出勤班次。
- 根据当月排班班次自动累加每个人的月总系数。
- 护士长参与月出勤班次统计，但月总系数显示为“护士长绩效单独核算”。
- 月表打印和 PDF 输出在整月排班明细下方增加“月度汇总”表。
- 首页主工作台继续保持当前周排班和周统计，不新增常驻月统计面板。

## 非目标

- 不计算月满勤标准。
- 不计算月加班班次。
- 不做奖金分配。
- 不做月结锁定。
- 不新增账号登录、角色体系或审计日志。
- 不调整现有周统计公式。

## 核算口径

月度汇总按班次口径计算，不按自然日工时计算。

统计范围为当前月表打印的全部日期。例如打印 2026-06 月表时，统计范围为 `2026-06-01` 至 `2026-06-30`。

对每个人：

- 只统计日期落在当前月表范围内的排班记录。
- 每条排班记录最多两个班次，两个班次分别参与统计。
- 班次不存在或已禁用时，不参与出勤和系数统计。
- `countsAttendance = true` 的班次计入月出勤班次。
- 所有启用班次的 `coefficient` 计入月总系数，包括不计出勤但有系数的班次。
- 护士和文员正常显示月总系数。
- 护士长月总系数不显示数字，显示“护士长绩效单独核算”。

月总系数保留两位小数，沿用周统计的四舍五入口径。

## 数据模型

新增月度汇总类型，放在现有领域类型文件中，结构与周统计保持相近但不包含满勤和加班字段。

```ts
export interface MonthlyStaffSummary {
  staffId: string;
  staffName: string;
  staffType: StaffType;
  attendanceShifts: number;
  coefficientTotal: number | null;
  coefficientExcludedReason: string;
}

export interface MonthlySummary {
  monthStart: string;
  monthEnd: string;
  totalDays: number;
  holidayNames: string[];
  rows: MonthlyStaffSummary[];
}
```

人员显示范围与月表排班明细一致：

- 启用人员始终显示。
- 已停用但当月有历史排班记录的人员显示，并标注停用历史。
- 已停用且当月无排班记录的人员不显示。

## 组件与数据流

新增 `calculateMonthlySummary(data, days)`：

- 输入当前应用数据和月表日期数组。
- 依据日期数组计算月开始、月结束、总天数和节假日名称。
- 复用现有班次配置和人员排序。
- 返回 `MonthlySummary`。

`App.vue` 中基于 `printMonthDays` 计算 `monthlySummary`，并传给 `PrintViews`。

`PrintViews.vue` 中月表结构调整为：

1. 标题与月份说明。
2. 节假日说明。
3. 整月排班明细表。
4. “月度汇总”表。

周表继续使用现有 `WeeklySummary`，不受月度汇总影响。

## 月表汇总打印格式

月表下方新增标题“月度汇总”。

汇总表字段：

- 人员
- 人员类型
- 月出勤班次
- 月总系数
- 备注

显示规则：

- 护士显示为“护士”。
- 文员显示为“文员”。
- 护士长显示为“护士长”。
- 普通人员月总系数显示两位小数。
- 护士长月总系数显示“单独核算”，备注显示“护士长绩效单独核算”。
- 停用历史人员在人员列中保留“停用历史”标记。

## 错误处理

月度汇总不新增服务端写入行为，因此不需要新的 API 错误处理。

当班次配置被停用或缺失时，月度汇总与周统计保持一致：忽略该班次，不计出勤，不计系数。

当月没有任何排班记录时，启用人员仍显示，月出勤班次为 `0`，月总系数为 `0.00`；护士长仍显示“单独核算”。

## 测试范围

新增或扩展以下测试：

- 月度计算单元测试：验证护士、文员、护士长的月出勤和月总系数。
- 月度计算边界测试：验证停用历史人员、有禁用班次、跨月记录不被统计。
- 月表打印组件测试：验证月表包含“月度汇总”和对应人员数据。
- 应用层测试：验证月表打印预览向 `PrintViews` 传入月度汇总。
- 完整测试与构建：`npm run test`、`npm run build`。

## 后续阶段

本阶段完成后，后续按以下顺序推进：

1. 奖金分配：基于月度汇总输入奖金总额并计算个人分配金额。
2. 月结锁定：锁定已确认月份，防止历史排班被直接修改。
3. 账号与角色权限：替代单一管理密码。
4. 审计日志：记录关键配置、排班和结算操作。
5. 正式部署：提供生产构建、后台服务、反向代理和日志策略。
