# 删除排班页排班提醒设计

日期：2026-06-21

## 背景

当前排班页会在工作台中展示“排班提醒”面板，持续列出当前周未排班、异常双班、缺失班次、停用班次引用和停用人员排班等提示。该功能虽然能辅助检查，但在日常排班页面上信息量偏大，容易造成视觉噪音。

系统同时还有“月结前数据检查”能力，会在确认月结前弹窗提示同类风险。该检查发生在关键操作前，不长期占用首页空间，应继续保留。

## 目标

删除排班页中的“排班提醒”功能，使排班工作台更简洁。

保留确认月结前的数据检查能力，继续在月结关键操作前提示风险。

## 范围

删除内容：

- 排班页“排班提醒 X 项”面板。
- 排班提醒列表、隐藏提醒数量和点击提醒打开排班格的交互。
- `App.vue` 中仅服务于排班页提醒的计算、类型导入和 helper。
- `src/lib/schedule-anomalies.ts` 及其测试。
- `src/styles/main.css` 中 `.schedule-anomaly-*` 样式。
- `App.test.ts` 中只验证排班提醒面板的测试。
- `docs/功能跟进清单.md` 中“当前周排班异常提醒”已完成功能描述。

保留内容：

- `src/lib/settlement-checks.ts` 和相关测试。
- `App.vue` 中确认月结前的 `月结前数据检查` 弹窗。
- 月结前检查提示的未排班、出勤不足、异常双班、缺失班次、停用班次引用和停用人员排班。

## 设计

前端排班页不再计算 `weeklyScheduleAnomalies`，也不再渲染提醒面板。排班页只保留日期选择、班次画笔、效率工具和排班表格。

月结与奖金流程不变。用户点击确认月结时，系统仍通过 `calculateSettlementChecks()` 做数据检查。如果存在问题，继续弹出“月结前数据检查”确认框，用户确认后仍可继续月结。

删除 `schedule-anomalies` helper 后，排班页不会再为当前周自动生成提醒；相关风险检查统一沉淀在月结前检查中。

## 测试策略

- 更新 `src/App.test.ts`：
  - 删除“显示当前周排班异常提醒并从提醒打开排班格”的测试。
  - 保留月结前数据检查相关测试。
- 删除 `src/lib/schedule-anomalies.test.ts`。
- 运行聚焦测试：

```bash
npm run test -- src/App.test.ts src/lib/settlement-checks.test.ts
```

- 运行全量验证：

```bash
npm run lint
npm run test
```

如全量测试在沙箱中因 `server/routes.test.ts` 本地监听触发 `listen EPERM`，按既有流程用非沙箱权限重跑同一条测试命令。

## 验收

1. 打开排班页，不再看到“排班提醒 X 项”面板。
2. 排班表格、班次画笔、复制上一周、批量设置和批量清空仍可正常使用。
3. 确认月结时，如存在排班风险，仍会弹出“月结前数据检查”。
4. `docs/功能跟进清单.md` 不再将“当前周排班异常提醒”列为已完成功能。
