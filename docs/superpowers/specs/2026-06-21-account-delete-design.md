# 删除账号设计

## 背景

系统已经具备账号维护、人员绑定、可管理人员范围、会话和审计日志能力。目前账号只能新增、修改或停用，测试账号和误建账号无法从页面删除。该设计补充“删除账号”能力，用于清理未使用或测试账号，同时保护正式账号和审计追溯。

本阶段只删除登录账号，不删除人员档案、排班、月结和审计日志。

## 已确认决策

- 在“系统配置 > 账号”中增加“删除账号”按钮。
- 只建议删除误建账号、未使用账号或测试账号。
- 正式账号、离职账号优先使用“停用”，不要物理删除。
- 删除账号时自动清理登录会话、账号绑定人员和可管理人员关系。
- 审计日志不删除，保留历史用户名、操作摘要和删除动作记录。
- 不允许删除当前登录账号。
- 不允许删除默认 bootstrap 管理员账号，例如默认 `admin`。
- 不允许删除启用中的账号，必须先停用后再删除。
- 不允许删除后导致系统没有启用的系统管理员。

## 产品行为

账号表单选中既有账号后显示“删除账号”按钮。新增账号草稿不显示删除按钮。

点击删除时弹出确认提示：

```text
确认删除该账号？仅建议删除误建或测试账号。删除后会清理登录会话、人员绑定和可管理人员关系，审计日志将保留。
```

删除成功后：

- 账号列表刷新。
- 当前账号表单重置为新增账号状态。
- 审计日志记录 `user.delete`。
- 如果被删账号曾在其他浏览器登录，其会话同步失效。
- 被删账号原本绑定的人员可以重新绑定给其他账号。

删除失败时，页面展示后端返回的错误信息。

## 删除规则

后端以管理员权限为准，只允许 `admin` 调用删除接口。

目标账号必须满足：

- 账号存在。
- 目标账号不是当前登录账号。
- 目标账号不是默认 bootstrap 管理员账号。
- 目标账号已停用。
- 删除后仍至少存在一个启用的 `admin`。

不把审计日志作为硬性阻断条件，原因是测试账号也可能产生登录、查看或保存测试数据的审计记录。审计日志会保留用户名和摘要，删除本身也会记入审计，从而兼顾测试清理和追溯。

## API 设计

新增接口：

```text
DELETE /api/users/:id
```

权限：

- 仅 `admin` 可访问。

成功响应：

```json
{ "ok": true }
```

失败响应：

- `404`：账号不存在。
- `400`：不能删除当前登录账号。
- `400`：默认管理员账号不能删除。
- `400`：请先停用账号后再删除。
- `400`：至少需要保留一个启用的系统管理员。

删除成功后记录审计：

```text
action: user.delete
targetType: user
targetId: <deleted user id>
summary: 删除账号：<username>，显示名：<displayName>，角色：<role>
```

## 存储设计

扩展 `AuthStore`：

```ts
deleteUser(input: { userId: string; actorUserId: string; bootstrapUsername: string }): Promise<AuthUser>
```

返回被删除账号的安全快照，用于记录审计摘要。

SQLite 删除操作在事务中完成：

1. 读取目标账号并校验删除规则。
2. 删除目标账号的 `user_sessions`。
3. 删除目标账号自己的 `user_managed_staff` 管理范围。
4. 将其他管理范围记录中的 `created_by = targetUserId` 置为 `null`。
5. 删除 `users` 中的目标账号。

说明：

- `audit_logs` 没有外键到 `users`，且已保存 `username` 文本，因此不需要修改历史审计。
- `user_sessions.user_id` 引用 `users.id`，删除用户前必须清理会话。
- `user_managed_staff.user_id` 有级联删除，但仍显式删除，便于内存实现和测试保持一致。
- `user_managed_staff.created_by` 引用 `users.id`，删除用户前必须置空，避免外键阻止删除。

内存 AuthStore 保持同样语义：

- 删除用户 map 中的目标账号。
- 删除该账号的所有会话。
- 清空其他内存关系中由该账号产生的创建者引用；如果内存结构没有保存创建者，则不需要额外处理。

## 前端设计

API 客户端新增：

```ts
deleteUser(id: string): Promise<{ ok: true }>
```

`ManagementDrawer`：

- emits 增加 `deleteUser`。
- 账号表单操作区增加危险按钮“删除账号”。
- 仅选中既有账号时显示。
- 使用 `el-popconfirm` 二次确认。
- 保存中或非管理员编辑模式时禁用。

`App.vue`：

- 增加 `handleDeleteUser`。
- 调用 `deleteUser` 后刷新账号列表。
- 删除成功后提示“账号已删除”。
- 删除失败展示后端错误。

## 错误处理

关键删除规则只在后端强制执行，前端不重复维护复杂判断。这样可以避免页面状态过期时绕过规则。

建议错误文案：

- `账号不存在`
- `不能删除当前登录账号`
- `默认管理员账号不能删除`
- `请先停用账号后再删除`
- `至少需要保留一个启用的系统管理员`

## 测试计划

后端测试：

- admin 可以删除已停用的测试账号。
- 删除账号会清理该账号会话。
- 删除账号会清理该账号的可管理人员关系。
- 删除账号会将其他关系中的 `created_by` 置空。
- 删除账号不会删除审计日志。
- 删除当前登录账号失败。
- 删除启用账号失败。
- 删除默认 `admin` 失败。
- 删除最后一个启用管理员失败。
- 删除不存在账号返回 404。
- 删除成功记录 `user.delete` 审计。

前端测试：

- 账号页选中既有账号后显示“删除账号”按钮。
- 新增账号草稿不显示删除按钮。
- 点击确认后 emit `deleteUser`。
- App 调用删除 API 后刷新账号列表。
- 删除失败时显示错误提示。

文档测试：

- 更新功能跟进清单，记录账号删除能力。
- 在使用说明中补充：正式账号建议停用，删除仅用于误建或测试账号。

## 非目标

本阶段不实现：

- 批量删除账号。
- 删除人员档案。
- 删除或改写历史审计日志。
- 自动识别账号是否“测试账号”。
- 恢复已删除账号。
