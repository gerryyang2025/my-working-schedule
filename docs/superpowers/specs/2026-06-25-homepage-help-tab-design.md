# 首页信息降噪与使用说明 Tab 设计

## 背景

当前首页在排班表上方同时展示系统标题、当前周范围、快速上手、核算规则、工具栏、账号权限提示和排班工作区。截图中第一屏的说明性内容占据较多空间，排班人员进入系统后的主要任务是查看和维护排班，过多说明会挤压排班表和画笔区。

本次优化目标是把“说明性内容”从首页操作区移到独立的使用说明页面，让首页更聚焦排班操作。

## 目标

- 首页第一屏减少非操作信息，排班表和画笔区更靠前。
- 新增一个“使用说明”工作台 tab，集中放置快速上手、人员权限和核算规则。
- 当前账号身份放到页面右上角，用户能清楚知道自己以什么身份登录。
- 删除页面标题区域右上角重复的周范围展示。
- 保留工具栏里和日期切换相关的周范围，因为它直接服务于排班操作。

## 范围

包含：

- `App.vue` 工作台 tab 新增“使用说明”。
- 将首页顶部的“快速上手”和“核算规则”迁移到“使用说明”页面。
- 将绿色账号权限提示迁移到“使用说明 > 人员权限”。
- 移除首页标题区右上角的 `selectedWeek.start 至 selectedWeek.end` 展示。
- 将当前登录用户信息移动到页面 header 右上角。
- 调整 toolbar，使其不再显示当前用户身份。
- 更新相关单元测试和样式测试。

不包含：

- 不改变登录、权限判断、可编辑人员范围或账号绑定逻辑。
- 不改变排班、查询、周统计、月结与奖金的业务逻辑。
- 不改变班次系数、满勤、加班或护士长单独核算规则。
- 不改后端 API、SQLite schema、数据迁移或部署脚本。

## 页面结构

### Header

Header 左侧继续显示：

- `国际医学部`
- `护理排班管理系统`

Header 右侧显示当前账号身份，例如：

```text
admin · 系统管理员
```

如果用户显示名和角色名不同，继续使用现有口径，例如：

```text
排班负责人 · 排班管理员
```

Header 不再显示当前周范围。当前周范围只保留在工具栏日期切换区域。

### Toolbar

工具栏保留：

- 日期选择。
- 上一周、本周、下一周。
- 当前周范围。
- 修改密码。
- 配置。
- 打印周表。
- 打印月表。
- 全屏。
- 退出登录。

工具栏不再显示当前账号身份，避免和 header 重复。

### Workbench Tabs

工作台 tab 调整为：

```text
排班
查询
周统计
月结与奖金
使用说明
```

默认仍进入“排班”。“使用说明”是普通工作台页面，不影响其他 tab 的状态。

## 使用说明页面内容

“使用说明”页面使用全宽正文布局，分为三个区块。

### 快速上手

说明日常排班操作：

- 通过日期选择或上一周、本周、下一周定位自然周。
- 在“排班”页选择画笔班次，再点击表格格子快速填班。
- 点击格子可进入单元格编辑，处理一人一天多个班次或备注。
- 搜索人员只影响当前页面展示，不改变排班数据。
- 复制上一周、批量休息、批量办公、批量清空只作用于当前可编辑范围。

### 人员权限

说明不同账号角色的可见和可操作范围：

- 系统管理员：可查看全科排班，可维护人员、班次、节假日、账号、排班和月结。
- 排班管理员：可查看全科排班，只能维护账号可管理人员范围内的排班和月结。
- 只读查看：可查看排班、查询、周统计和月结结果，不能保存修改。
- 绑定人员只用于标识账号本人，不会自动授予排班权限；编辑范围由账号可管理人员决定。

首页不再单独显示绿色账号权限提示。

### 核算规则

保留当前核算说明，并移动到使用说明页：

- 按班次而不是自然日计出勤。
- 满勤默认 5 个班次。
- 影响满勤的节假日会扣减满勤标准。
- 加班 = `max(0, 出勤班次 - 满勤标准)`。
- 总系数按班次系数累加。
- 护士长绩效系数单独核算。
- 班次系数列表继续读取当前启用班次配置展示。

## 组件边界

`AppToolbar` 继续负责日期切换和操作按钮，但不再接收或渲染 `currentUser`。当前用户身份由 `App.vue` header 渲染。

`App.vue` 负责：

- 新增 `help` tab。
- 渲染使用说明内容。
- 复用现有 `shiftCoefficientDescriptions`。
- 复用当前用户信息展示口径，避免 `AppToolbar` 和 header 出现两套不同格式。

可以在 `App.vue` 内新增轻量 computed，例如 `currentUserIdentityLabel` 和 `currentUserRoleLabel`。如果后续还有更多页面需要用户身份文案，再考虑抽出独立 helper；本次不新增跨模块抽象。

## 样式设计

首页去掉 `app-info-panel` 和 `admin-mode-banner` 后，工作台整体会上移。新的“使用说明”页采用白底边框内容区，区块之间使用清晰标题和紧凑段落，风格与现有后台工具界面保持一致。

Header 右侧的用户身份使用和当前 `.toolbar-user` 类似的轻量标签样式，但命名应贴合 header，例如 `.header-user`。移动端 header 可换行，用户身份标签在标题下方或右侧自然折行，不遮挡标题。

保留工具栏周范围 `.toolbar-week-range`，因为它与日期选择和周切换在同一上下文中。

## 验证口径

需要覆盖以下场景：

- 首页不再渲染 `.app-info-panel`。
- 首页不再渲染 `.admin-mode-banner`。
- Header 不再渲染独立的周范围 chip。
- Header 右上角渲染当前账号身份。
- Toolbar 不再渲染当前账号身份，但仍显示工具栏周范围。
- 工作台 tab 包含“使用说明”，默认仍选中“排班”。
- 点击“使用说明”后显示快速上手、人员权限、核算规则和班次系数说明。
- 系统管理员、排班管理员、只读查看的权限说明在使用说明页可见。
- 现有排班、查询、周统计、月结与奖金 tab 切换行为保持不变。

## 风险与取舍

这次调整会让首次进入系统时的说明不再直接显示在首页。取舍是：首页更清爽，但新用户需要点击“使用说明”查看规则。由于这是内部排班工具，日常高频用户更需要减少干扰，因此推荐把说明集中到独立 tab。

当前账号权限提示从首页移走后，用户仍能通过 header 身份标签和“使用说明 > 人员权限”理解自己的角色。真正的权限控制仍由现有逻辑执行，本次只是改变说明位置。
