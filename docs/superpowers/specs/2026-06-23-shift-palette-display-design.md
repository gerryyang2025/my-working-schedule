# 首页画笔班次显示优化设计

## 背景

首页排班画笔当前把所有启用班次横向排列在一行。班次类型增加后，用户需要横向滚动才能查看全部画笔，查找和选择效率下降。此次优化只改首页画笔区的展示方式，不新增班次分类字段，不调整系统配置、排班数据结构、统计规则或打印规则。

## 目标

- 班次较多时，首页画笔区能直接看到常用和普通班次，减少横向滚动。
- 常用班次优先展示，普通班次跟随展示。
- A、P、N 系列在画笔区有稳定的系列颜色，方便快速辨识。
- 保持现有画笔点击填班交互不变。

## 范围

包含：

- 首页排班 tab 内的 `ShiftPalette` 展示优化。
- 启用班次的前端分组、固定顺序和兜底排序。
- 画笔区桌面端与移动端样式调整。
- `ShiftPalette` 组件测试和必要的样式测试。

不包含：

- 不新增“常用 / 普通”班次配置字段。
- 不修改 SQLite schema、API 类型或服务端存储。
- 不改变系统配置里的班次管理表单。
- 不改变排班表单元格、打印、统计和月结逻辑。

## 展示设计

画笔区从单行横向列表改为两块分区：

- 第一块标题为“常用”。
- 第二块标题为“普通”。

每个分区内部使用紧凑按钮并自动换行。按钮仍显示班次简称，点击行为仍然是选中该班次作为快速填班画笔。当前选中的画笔继续用高亮状态表达。

桌面端不再依赖横向滚动查看全部班次。移动端保留 sticky 画笔区，常用和普通两组都直接显示，按钮自然换行。为了避免挤占过多排班表空间，按钮尺寸保持当前紧凑风格，不做折叠展开。

如果某个分区没有启用班次，则不显示该空分区。禁用班次不出现在画笔区。

## 分组与排序

本阶段不引入持久化分类字段，分组由前端根据班次简称或名称识别。

常用班次固定顺序：

```text
常班、A1组长、A2、A3组长、A4、A5、A6、A7、P1、P2、P3、N1、N2、办公、休、带检
```

识别时以班次简称优先，名称作为补充。例如简称为 `A1` 或名称包含 `A1组长` 都归入常用；简称为 `办公` 或名称包含 `办公` 都归入常用。常用组内部优先按上述固定顺序展示。

普通班次固定顺序：

```text
护理总值、进修、备1、备2、培训、公休、婚假、育儿假、病假、产假、事假、丧假、产假/休、保健
```

不匹配常用固定名单的启用班次进入普通组。普通组内部优先按上述固定顺序展示；未命中固定名单的新增班次排在普通组末尾，并按原 `sortOrder`、再按 `id` 稳定排序。

## 颜色规则

画笔区增加系列兜底色：

- A 系列统一使用蓝色。
- P 系列统一使用绿色。
- N 系列统一使用红色。

A、P、N 系列只影响首页画笔按钮的显示色。非 A/P/N 班次继续使用班次配置中的 `color`。排班表单元格、打印、统计和月结仍使用现有班次配置颜色，不随本次画笔区兜底色改变。

## 组件边界

`ShiftPalette` 继续只接收 `shifts` 和 `selectedShiftId`，继续通过 `select` 事件向父组件报告选中的班次。分组、排序和画笔区颜色派生留在组件内部，避免影响其他排班模块。

父级 `App.vue` 不需要新增状态。现有 `selectedShiftId` 选择流和 `ScheduleGrid` 快速填班逻辑保持不变。

## 验证口径

需要覆盖以下场景：

- 启用班次按“常用 / 普通”两组显示。
- 常用组按固定顺序显示 A、P、N、办公、休、带检等班次。
- 普通组按固定顺序显示护理总值、进修、备班、假别等班次。
- 未命中固定名单的新班次进入普通组末尾，并按原排序稳定显示。
- 禁用班次不显示。
- A、P、N 系列在画笔区分别使用统一颜色。
- 选中态和点击 `select` 事件保持现有行为。
- 样式不再要求单行横向滚动，分区内按钮可以换行。

## 风险与取舍

前端规则识别比持久化分类轻量，能快速解决首页浏览问题，但它依赖班次简称和名称的稳定性。若未来需要管理员自由指定分类，应另起需求增加班次配置字段和数据迁移。

本阶段选择直接展示两组，不做折叠展开。这样普通班次也能一眼看到，代价是画笔区高度会比当前单行略高；通过紧凑按钮和自动换行控制占用空间。
