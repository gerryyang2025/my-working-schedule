
# Web 版本实现方案

## 1. 后端架构：Go + Gin 框架
*   **高效接口开发**：利用 **Gin** 框架轻量级、高性能的特点，快速构建 RESTful API。针对排班系统频繁的“点击即输入”交互，Gin 的高性能路由能确保极低的响应延迟。
*   **核心逻辑实现**：
    *   **核算引擎**：在 Go 逻辑层编写**出勤与绩效核算算法**。根据来源 的规则，系统将自动识别法定节假日（如端午节），动态调整该周的“满勤”阈值（如从 5 天降至 4 天），并实时计算每位员工的系数盈亏。
    *   **并发处理**：利用 Go 的 Goroutines 处理月度奖金的总额分配计算，确保在处理多人、多班次汇总时保持系统流畅。
*   **接口设计**：
    *   `/api/v1/staff`：管理护士长、护士及文员名单（对应来源 中的工号与姓名）。
    *   `/api/v1/schedule`：处理每日排班的增删改查。
    *   `/api/v1/calculation`：执行周/月绩效汇总及奖金分配计算。

## 2. 数据库存储：etcd (K/V 存储)
由于数据量较小，使用 **etcd** 作为主存储方案是可行的，它能提供极高的可靠性和一致性。
*   **Key 结构设计建议**：
    *   **人员配置**：以 `/staff/{job_id}` 为 Key，存储人员姓名、职级及预设系数（如护士长 1.8，组长 1.5）的 JSON 数据。
    *   **排班记录**：以 `/schedule/{year-month}/{day}/{job_id}` 为 Key，存储当日班次（如 A1, P1, N1）。这种结构便于按日期范围快速检索。
    *   **配置参数**：以 `/config/shifts` 存储所有常用与不常用班次的定义及对应的颜色标识（如“夜”显示为红色）。
*   **数据持久化**：利用 etcd 的快照机制确保排班数据安全，防止因系统故障导致的数据丢失。

## 3. 前端交互与视觉优化
*   **动态表格渲染**：前端（Vue/React）通过 API 获取 etcd 中的 Key-Value 数据，动态生成排班矩阵。
*   **视觉辅助功能**：
    *   **颜色标识**：根据后端返回的班次类型，自动为“夜”、“价”、“休”等特殊状态标注红色，并为周末及法定节假日背景加红。
    *   **全屏与打印**：前端实现一键全屏显示，并提供优化的打印视图，支持护士长直接打印排班表申请。

## 4. 自动化核算流程实现
*   **周汇总触发**：系统根据当前日期判断是否为周五，自动触发后端 Go 服务的核算函数，从 etcd 提取该周数据，计算“出勤总天数”和“总系数”，并对照满勤规则得出 +/- 值。
*   **月核算与奖金分配**：在每月最后一天，系统汇总结清全月数据。管理员在 Web 界面输入奖金总金额后，Go 服务将遍历 etcd 中的人员总系数，实时返回各工号应得的奖金数额。

# 方案总结

采用 **Go + Gin + etcd** 的架构方案，不仅能满足您对系统轻量化、高性能的要求，还利用了 Go 强大的逻辑处理能力来实现复杂的排班核算规则。这种配置非常适合科室内部的小型化、高可靠系统部署。
